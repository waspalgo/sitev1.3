PROJECT OVERVIEW
You are building the official website for WASPALGO, a professional algorithmic trading technology brand.
The core product is an internal algorithmic engine called WA-AMIR:
WA-AMIR = WASP Autonomous Market Intelligence Reactor.

There are two variants of WA-AMIR:
- WA-AMIR ST V1 (Short Term) – available now.
- WA-AMIR LT V1 (Long Term) – planned and currently in internal testing (not yet accessible to clients).

The website must look like a serious, premium, AI-powered fintech / quant trading brand:
- Not like a retail “get rich quick” landing page.
- Tone = calm, professional, transparent, technical.

The target language for the user interface is French by default. English can be added later via a language switcher, but the main content must be in French.

TECH STACK
- Use Next.js App Router (latest) with TypeScript.
- Use TailwindCSS for all styling (no CSS frameworks other than Tailwind).
- Use a mobile-first, fully responsive approach for phones, tablets, laptops and large screens.
- Organize reusable UI as React components under /components.

BRAND & POSITIONING
- WASPALGO = algorithmic trading technology brand, not a regulated investment manager.
- WA-AMIR is presented as an algorithmic engine that connects to the user’s own broker account.
- The user always keeps control of their capital and account; the algorithm simply automates decisions following predefined rules.
- The site must never:
  - promise profits,
  - guarantee returns,
  - or suggest “safe gains”.
- You may describe:
  - logic (short term vs long term),
  - exposure (XAUUSD vs BTC),
  - backtest methodology and stress tests,
  - risk warnings.
- Always pair any mention of performance or backtests with a clear risk disclaimer in French.

RESPONSIVE DESIGN REQUIREMENTS
- The layout must be **mobile-first** and fully responsive.
- On small screens (phones ~360–430px width):
  - Navigation collapses into a hamburger menu.
  - Sections are stacked vertically with generous spacing.
  - Text remains readable: limit line-length, adapt font sizes, and add comfortable padding.
  - Cards use full width with margin between them.
  - Charts or “fake dashboards” must never overflow horizontally: always respect max-width: 100%.
- On medium screens (tablets ~768px):
  - Use 2-column layouts for hero (text + dashboard preview) and some sections.
  - Cards can be displayed as 2-column grids.
- On large screens (~1024–1920px+):
  - Use 2–3-column grids where appropriate (e.g. “Nos algorithmes”, “Market snapshot”).
  - Hero can be a 2-column layout: text left, fake dashboard right.
- Test visually and ensure everything looks polished from mobile to large desktop.

  - Indiquer que lors de ce type de choc sur les cryptomonnaies, les flux se sont historiquement souvent redirigés vers l’or, ce qui, dans les simulations, a permis à l’algorithme de bénéficier de ce mouvement sur XAUUSD.
